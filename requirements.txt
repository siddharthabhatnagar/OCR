# PyMuPDF4LLM — lightweight pure-Python PDF-to-markdown for RAG.
# Total bundle ~30 MB (vs Docling's ~5 GB).
# No torch, no transformers, no model weights — just PDF parsing.
#
# Why PyMuPDF4LLM over Docling on Vercel:
#   - Bundle: 30 MB vs 5 GB (Vercel limit is 225 MB)
#   - Cold start: <1 sec vs 5 sec
#   - Per-page: <1 sec vs 2-5 sec
#   - Tables: good (rule-based) vs excellent (ML)
#   - Equations: basic vs basic
#   - Cost: $0 forever vs $0 forever
#
# Trade-off: PyMuPDF4LLM uses rule-based table detection (no ML model),
# so complex tables with merged cells may be less accurate than Docling.
# For most RAG use cases (papers, reports, manuals) it's more than enough.

# IMPORTANT: pin pymupdf4llm to a version BEFORE it started pulling in
# pymupdf-layout (which drags in onnxruntime + networkx + numpy, ~200MB
# of ML deps for layout detection you don't need). Newer pymupdf4llm
# (>=1.x) resolves to that heavy stack even though the code here never
# touches it. This pin keeps the bundle ~90-100MB, well under Vercel's
# 225MB function limit.
pymupdf4llm==0.0.17
PyMuPDF>=1.24.0,<1.28.0
Pillow>=10.0.0

# Web framework for the ASGI app
starlette>=0.37.0
